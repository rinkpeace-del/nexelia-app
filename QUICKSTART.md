# NEXELIA クイックスタートガイド（VS Code）

## 🚀 3ステップで開始

### Step 1: ターミナルで認証サーバーを起動（20秒）

VS Code で新しいターミナルを開きます：
```
Ctrl + Shift + `
```

以下のコマンドを実行：
```bash
node auth-server.js
```

**✅ 成功時の表示：**
```
認証サーバーが起動しました: http://localhost:5001
ユーザーデータファイル: C:\Users\...\NEXELIA\users.json
```

### Step 2: Live Server を起動（10秒）

別のターミナルで、または VS Code UI から Live Server を起動：

**オプション A：コマンドパレットから（推奨）**
1. `Ctrl + Shift + P` を押す
2. `Live Server: Open with Live Server` と入力
3. Enter を押す

**オプション B：右クリック**
1. index.html を右クリック
2. "Open with Live Server" を選択

### Step 3: ブラウザで動作確認（30秒）

自動的にブラウザが開きます（`http://localhost:5500`）：

1. **ログインページが自動表示**
2. **「ユーザー登録」** をクリックして新規登録、またはテストユーザーでログイン
   - ユーザー名: `testuser`
   - パスワード: `test1234`
3. **✅ ログイン成功** → レッスン一覧ページが表示される
4. ページ上部にユーザー名が表示される

## ⚠️ よくある問題と解決方法

### 「ネットワークエラーが発生しました」と表示される

**原因：** auth-server が起動していない

**解決：**
```bash
# ターミナルで以下を実行
node auth-server.js
```

→ "認証サーバーが起動しました" と表示されることを確認

---

### ログインページが表示されない

**原因：** localStorage にキャッシュが残っている

**解決：**
1. ブラウザの開発者ツール（F12）を開く
2. "Application" または "Storage" タブを選択
3. "Local Storage" → "http://localhost:5500" を選択
4. 全て削除
5. ページリロード（F5）

---

### Live Server が起動しない

**原因：** ポート5500が他のアプリで使用されている

**解決：**
- VS Code は自動的に次のポート（5501, 5502など）を試します
- ブラウザのアドレスバーで実際のポート番号を確認してアクセス
