# NEXELIA 認証システム セットアップガイド

## 概要

NEXELIA には、ユーザー登録とログイン機能を備えた認証システムが実装されています。
このドキュメントでは、VS Code Live Server で開発する際に、認証システムを正常に動作させる方法を説明します。

## 必須要件

- Node.js がインストールされていること
- VS Code の Live Server 拡張機能がインストールされていること

## セットアップ手順

### 1️⃣ ターミナルを開く

VS Code で新しいターミナルを開きます：
```
Ctrl + Shift + `（バッククォート）
```

### 2️⃣ 認証サーバーを起動

NEXELIA ディレクトリで、以下のコマンドを実行します：

```bash
node auth-server.js
```

**期待される出力：**
```
認証サーバーが起動しました: http://localhost:5001
ユーザーデータファイル: C:\Users\rinkp\OneDrive\デスクトップ\NEXELIA\users.json
```

### 3️⃣ VS Code Live Server を起動

別のターミナルまたはVS Code内で、Live Serverを起動します：

**方法 A: コマンドパレットから起動**
1. `Ctrl + Shift + P` を押す
2. "Live Server: Open with Live Server" と入力
3. Enter を押す

**方法 B: ファイルを右クリック**
1. index.html を右クリック
2. "Open with Live Server" を選択

**方法 C: ステータスバーから起動**
- VS Code 下部のステータスバーで "Go Live" をクリック

### 4️⃣ ブラウザでアクセス

自動的にブラウザが開きます（通常は `http://localhost:5500`）。
もし自動で開かない場合は、ブラウザで `http://localhost:5500` にアクセスしてください。

## ログイン/登録フロー

### 新規ユーザー登録

1. ブラウザで `http://localhost:5500` にアクセス
2. localStorage が空の場合、自動的に **ログインページ** にリダイレクト
3. 「アカウントをお持ちでないですか？」→ **「ユーザー登録」** をクリック
4. ユーザー登録フォームで以下を入力：
   - **ユーザー名**：3文字以上
   - **パスワード**：6文字以上
   - **パスワード（確認）**：上記と同じ
5. **「アカウント作成」** ボタンをクリック
6. 登録成功時、自動的にログインページにリダイレクト

### 既存ユーザーでログイン

1. ログインページで以下を入力：
   - **ユーザー名**
   - **パスワード**
2. **「ログイン」** ボタンをクリック
3. ログイン成功時、**レッスン一覧ページ** にリダイレクト
4. ページ上部にユーザー名が表示される
5. **「ログアウト」** ボタンでログアウト可能

## トラブルシューティング

### ログイン/登録ページが表示されない場合

**原因：auth-server が起動していない**

解決方法：
1. VS Code のターミナルで `node auth-server.js` を実行してください
2. 以下のメッセージが表示されることを確認：
   ```
   認証サーバーが起動しました: http://localhost:5001
   ```

### "ネットワークエラーが発生しました" と表示される場合

**原因：auth-server が起動していない、またはポート5001が別のプロセスで使用されている**

解決方法：
1. `node auth-server.js` が実行中か確認
2. ポート5001が他のプロセスで使用されていないか確認：
   ```bash
   # Windows
   netstat -ano | findstr :5001

   # Mac/Linux
   lsof -i :5001
   ```
3. 競合がある場合は、そのプロセスを終了してから auth-server を再起動

### ログイン/登録後にレッスンページが表示されない場合

**原因：localStorage に不正なデータが存在している可能性**

解決方法：
1. ブラウザの開発者ツールを開く（F12）
2. 「Application」または「Storage」タブを選択
3. 「Local Storage」から `http://localhost:5500` を選択
4. 全てのエントリを削除
5. ページをリロード（F5）して再度ログイン

### ポート5500がすでに使用されている場合

VS Code Live Server は自動的に次のポートを試します（5501, 5502など）。
ブラウザのアドレスバーで表示されるポート番号を確認してアクセスしてください。

## API エンドポイント（参考）

認証サーバーは以下のエンドポイントを提供しています：

| メソッド | エンドポイント | 説明 |
|---------|-------------|------|
| POST | `/api/auth/register` | 新規ユーザー登録 |
| POST | `/api/auth/login` | ユーザーログイン |
| GET | `/api/auth/user/:userId` | ユーザー情報取得 |
| GET | `/api/auth/users` | 全ユーザー取得（管理用） |

```
├── lessons.html            # レッスン一覧ページ（ユーザー情報表示付き）
├── users.json              # ユーザーデータベース（自動生成）
└── AUTHENTICATION.md       # このファイル
```

## 開発時の注意点

1. **auth-server.js の起動は必須**
   - ローカル開発時は常に `node auth-server.js` を実行してください
   - 複数のブラウザセッションから同時にアクセス可能です

2. **users.json について**
   - 自動的に生成されるファイルです
   - 手動で編集しないでください
   - リセットする場合は、ファイルを削除して auth-server を再起動してください

3. **HTTPS について**
   - ローカル開発環境なので HTTP を使用しています
   - パスワードはSHA256ハッシュ化されて保存されています
   - 本番環境では HTTPS の使用を強く推奨します
